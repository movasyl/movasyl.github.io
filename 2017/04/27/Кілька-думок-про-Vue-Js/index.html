Vue.js is a framework that helps developers build web application. This is not the first framework that I encountered. I have a little bit experience with Handlebars and AngularJS. Handlebars is very easy to pick up. But there’s limitation when you want to bind the data, such as you can’t really do much computation when you use double curly brackets to bind your data in the front end. AngularJS separates the states and the controllers and you have to do much more set-up comparing to Handlebars.
I will talk about several things that I like about Vue.js here:
The guide for Vue.js 2.0 is very well written and includes a lot of useful examples. As a beginner to web application frameworks, reading original documentations is not the easiest task for me. But I found that most of the time I can follow the guide.
I like the idea about components. components are templates that includes its own html, scripts and styles. If you are developing a relatively large application, you might want to do it little by little. There might be a lot of repetitive part on every page. With components, you don’t have to rewrite or restyle it. You can separate and combine them as much (or little) as you like. You can use components inside other components, which make creating flexible layout much more easier. I have to admit that I was fascinated about the idea of components when I first knew about it. It makes your codes much more reusable. You can even swap components by just creating a “component” tag with a “is” attribute, or use “slot” tag in order to flexibly mount the component to a target div.
Data and event handling are easy. Just like AngularJS controllers, you can include data and method when you set up a component. And by using “props” and “events”, you can easily use the data and method of the parents or children components. That make data and event handling able to be done mostly in front end. When I work with Handlebars, I have to write most of the algorithms and logic in the backend.
Setting up can be pretty simple. If you just want to use it for a single part of your application. You just need to set up a script tag and open a new Vue instance. So I believe it’s very friendly for small applications.
Instance lifecycle hooks are very handy. You can set up lifecycle hooks for every Vue instance. They are like event listeners for special “life stage” of the instance. For example, you can do something when the instance is “created”, “mounted”, “updated” or “destroyed”. It’s sort of like AngularJS controllers. But they are specifically “lifecycle-oriented.”
There’s vue-cli available with existing project templates. It’s recommended by Vue.js official site and it’s easy to use. I also find a express-vue module for wiring Vue templates with Express.
I will learn React in the next week and I will be back and talk about the similarities and differences between them.
Thanks for reading.
